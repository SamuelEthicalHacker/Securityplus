
# Access Control Best Practices

Un control de acceso efectivo reduce la superficie de ataque y garantiza que los usuarios solo accedan a lo que necesitan. Las mejores prácticas claves incluyen:


1. Aplicar Principio de Mínimo Privilegio (PoLP)

Los usuarios deben tener solo los permisos necesarios para sus tareas. Esto reduce daños en caso de compromiso o abuso.

2. Implementar Segregación de Funciones (SoD)

Divide tareas críticas entre múltiples usuarios para evitar fraudes o uso indebido (ej.: quien crea pagos no debe aprobarlos).

3. Revisiones de Acceso Periódicas

Auditar permisos regularmente para eliminar accesos innecesarios, cuentas antiguas o privilegios excesivos.

4. Control de Acceso Basado en Roles (RBAC)

Estandarizar permisos por rol reduce errores humanos y simplifica la administración.

5. Aplicar MFA Siempre que Sea Posible

Añadir una capa adicional de seguridad más allá de usuario/contraseña.

6. Desactivar Cuentas Inactivas

Las cuentas no utilizadas representan puertas abiertas para atacantes.

7. Control de Acceso Basado en Atributos:

Aplicar restricciones por:

ubicación

horario

tipo de dispositivo

riesgo detectado

Al estilo Zero Trust.
